# Служба управления доставкой (Telegram-бот)

[![Python](https://img.shields.io/badge/Python-3.13-blue.svg)](https://www.python.org/)
[![Tests](https://img.shields.io/badge/tests-18%20passed-brightgreen.svg)]()

Проект курсовой работы по дисциплине «Программирование на языке высокого уровня».  
Реализована **бизнес-логика** службы доставки, которая может быть использована как основа для Telegram-бота.  
Взаимодействие с пользователем эмулируется через вывод сообщений в лог — это сделано намеренно, чтобы сконцентрироваться на чистой архитектуре и предметной области.

> **Важно:** Это не готовое коммерческое решение, а учебный проект, демонстрирующий принципы SOLID, Clean Architecture и паттерны проектирования на языке Python.

## Функциональность

- Создание заказов (обычная / экспресс-доставка)
- Назначение свободного курьера на заказ
- Отслеживание статуса заказа:
  - `ожидает` → `назначен` → `забран` → `доставляется` → `доставлен`
- Регистрация новых курьеров
- Ручное обновление статуса доставки (pickup → delivering → complete)

## Архитектура проекта

Проект построен по принципам **чистой архитектуры** (Clean Architecture) и разделён на слои:

| Слой | Назначение | Ключевые классы |
|------|------------|----------------|
| **Domain** | Сущности предметной области | `Order`, `Courier`, `OrderStatus`, `DeliveryType` |
| **Application** | Бизнес-логика и варианты использования | `DeliveryService`, `OrderRepository`, `CourierRepository` |
| **Infrastructure** | Внешние зависимости (эмуляция Telegram) | `TelegramBot` |
| **Host** | Точка входа, настройка зависимостей | `main()` |

## Используемые технологии и паттерны

- **Python 3.13** + asyncio
- **Pytest** + pytest-asyncio — модульное тестирование
- **Паттерны проектирования**:
  - Репозиторий (`OrderRepository`, `CourierRepository`)
  - Внедрение зависимостей (DI)
  - Стратегия (на примере типов доставки)
- **Принципы SOLID** и изоляция тестов через `unittest.mock`

## Быстрый старт

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/mizioooz/delivery-service-bot.git
   cd delivery-service-bot
   ```
   
2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv .venv
   source .venv/bin/activate      # Linux/macOS
   .venv\Scripts\activate         # Windows
   ```
   
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
   
4. Запустите эмуляцию работы бота:
   ```bash
   python main.py
   ```
   
## Тестирование

Проект покрыт модульными тестами (18 тестов). Для их запуска выполните:
   ```bash
   pytest -v
   ```

## Эмуляция команд Telegram-бота

В текущей версии бот не требует настоящего токена и не подключается к Telegram. Команды эмулируются вызовом соответствующих методов:

|        Метод        | Аналог команды |
|---------------------|----------------|
|handle_order_command|/order|
|handle_status_command|/status|
|handle_courier_command|/courier|
|handle_deliver_command|/deliver|

Это позволяет полностью протестировать бизнес-логику без внешних зависимостей.
